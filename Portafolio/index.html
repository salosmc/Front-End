TO-DO

API Dental

El objetivo de esta API es la gestión de turnos de una clínica dental , así como sus pacientes y odontólogos. Se desarrolló en Java, utilizando Spring Boot y JPA para la persistencia de datos en MySQL. También se aplicó JWT para la administración de peticiones de los usuarios logueados y Swagger para la documentación.

Servidor levantado en Heroku.

Front Digital Booking

Desarrollado en el marco de Digital House, se trabajó en equipo aplicando metodología SCRUM y cultura DevOps. El front se desarrolló con React nativo y se hizo uso de Hooks. También se usaron librerías y dependencias como lo son Sweetalert para las notificaciones, apis de google para el uso de mapas como también de aws para persistir imágenes en un Bucket de S3, entre otros.

Servidor habilitado de 14:00hs a 18:00hs (Arg), los dias de semana.

BackEnd DB

Desarrollado en el marco de Digital House, se trabajó en equipo aplicando metodología SCRUM y cultura DevOps. La API fue desarrollada en java utilizando SprintBoot y JPA para la persistencia de datos en MySQL. Entre otras cosas se hicieron llamadas nativas a Storede Procedure de MySQL, uso de Sprint Security, JWT HS256 y consulta a APIs de terceros para validar usuarios vía mail, entre otros.

Servidor habilitado de 14:00hs a 18:00hs (Arg), los dias de semana.
